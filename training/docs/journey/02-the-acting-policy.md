## A GPT that acts

How does the model run? Part 1 gave the shape — the sequence is `(state, action)`
pairs. Here is why that asymmetry *is* the policy. Causal GPT-RL is a **GPT that
acts**, and the two halves are *not* symmetric:

- the **state arrives from the world** at each step — you supply it;
- the **action is generated by the model** — and fed straight back in as the next
  input, exactly like a token.

That is the whole trick: **the action the model emits stays in the model.** You
receive it, but it also re-enters the context as history (kept in a KV-cache).
*Action generated, state given* — that asymmetry is the policy.

Because the action feeds itself, a **single model rolls out a whole episode** — no
critic, no value network, no auxiliary machinery at inference. You drive it by
alternating `act()` and `observe()`, and this is why an AR model is *used*
differently from a conventional RL policy.

---

## Run it

You now have everything to run the loop. A trained policy ships as a bundle; load
it and roll out. The runtime rebuilds the exact input/output shapes from the spaces
you declared in Part 1, and refuses any bundle it cannot serve rather than
mis-decoding.

Loading exposes the one knob the rollout has — **how much history it keeps**:

| Parameter | What it is | Can you set it? |
|---|---|---|
| `context_length` | the trained token window the model was built for | **No** — fixed in the bundle at training time |
| `kv_cache_max_len` | how much past the rollout retains | **Yes** — passed at load time; defaults to `4 × context_length` |

Then drive it by alternating `act()` and `observe()`:

```python
import gymnasium as gym
from causal_gpt_rl.inference import load_runner

env = gym.make("YourEnv-v0")
runner = load_runner("path/to/bundle", kv_cache_max_len=128)  # optional

obs, _ = env.reset()
runner.reset(obs)                 # seed the context with the first observation
action = runner.act()             # a valid sample of your action_space
# ... step the env, then runner.observe(next_obs), and repeat
```

For the exact per-space input/output contract at inference — what you pass and what
you get back for every space type — see [spaces.md](../../../docs/spaces.md).
